## 1. Memory function

This document exists only to answer:

- what persists
- how it persists
- what remains active memory
- what becomes inactive memory
- how memory is reconstructed into workflow
- how memory trust tier may rise without silently becoming canon
- how retained read-side artifacts may be reconstructed into workflow without silent authority uplift

Nothing else belongs here.

## 2. Core memory invariant

Memory must preserve inherited constraint-bearing identity across persistence.

Memory may:

- retain
- archive
- reconstruct
- raise memory trust tier lawfully

Memory must not:

- destroy
- replace
- or silently inflate

the active constraint-bearing identity inherited from earlier workflow verbs.

Memory may not silently convert trusted memory into canon.

Memory may not silently redefine evaluation outcome after evaluation closure.

## 3. Memory macro verb

Memory is the fifth macro verb:

- Admit
- Form

- Memorize

Memory emits exactly one bound object:

**Memory object**

If a memory object is not emitted, the pass must:

- continue within Memorize
- recruit downward
- defer
- or abort

## 4. Memory phases

Memory proceeds through:

- Reconstruct
- Retain
- Archive
- Promote

Memory may also receive deferred material from earlier verbs when lawful.

Defer is not itself a memory state.

Defer is a routing posture available from any macro verb.

---

## 5. Phase functions

### 5.1 Reconstruct

Reactivates or regenerates prior retained material into workflow-visible form.

Used when:

- prior state must be re-entered
- comparison requires regenerated memory-side structure
- an inactive or retained object must become active input again

Reconstruct must declare its source and lens.

Reconstruct is not truth by itself.

Reconstruct is the closest memory-side equivalent of remembering.

---

### 5.2 Retain

Keeps current momentum stable as active memory continuity.

Retain is used when:

- the object should persist
- the memory remains live enough to support future workflow
- the object is not being archived away from active continuity

Retain preserves usable continuity.

Retain does not imply canon.

---

### 5.3 Archive

Moves memory into non-active retained state.

Archive is used when:

- the object should persist
- but should not remain active working memory
- and should remain available for later reconstruction, audit, or reference

Archive is a memory state.

Archive is not workflow resolution by itself.

---

### 5.4 Promote

Raises memory trust tier inside memory space.

Promote is used when:

- a memory object has earned stronger bounded trust
- reuse conditions are explicit
- trust uplift is useful
- the uplift remains governable

Promote inside memory space is not automatically canon admission.

Canon promotion remains separately gated by higher boundary law.

---

## 6. Forbidden moves

Memory must not:

- silently rewrite evaluation outcome
- silently redefine structure
- collapse recurrence into truth
- collapse trusted memory into canon
- treat archive as deletion
- treat retain as proof
- reconstruct without declared source or lens
- silently widen reuse conditions

---

## 7. Memory-state routing

Memory answers:

- retain
- archive
- promote in memory space
- reconstruct into workflow when needed

Memory does not answer:

- immediate audit meaning
- immediate review significance
- immediate workflow acceptance or revision

Those belong to Evaluate.

---

## 8. Downward recruitment

Memory recruits downward when lawful persistence cannot be determined honestly.

### To Evaluate

When:

- immediate judgment is incomplete
- outcome significance is still unstable
- evaluation closure was premature or insufficient

---

### To Decide

When:

- the wrong candidate was realized
- memory persistence cannot be set because the decision object was flawed
- the retention target is invalid because the selected path was wrong

---

### To Form

When:

- the object cannot persist because identity structure is unstable
- the retained object is malformed
- the object boundary is mixed or unresolved

---

### To Admit

Only when:

- the original source or scope was malformed
- the governing policy made lawful memory impossible from the start

---

## 9. Memory object

Memory emits:

**Memory object**

This object contains:

- persistence posture
- retention or archive state
- reconstruction availability
- trust-tier status inside memory space
- declared reuse conditions

This object becomes the basis for:

- future reconstruction
- retained continuity
- bounded long-term memory posture
- possible later canon consideration under separate authority

## 10. Separation rule

Memory answers:

- what persists?
- how does it persist?
- what becomes inactive memory?
- what becomes trusted long-term memory inside memory space?

Memory does not answer:

- what happened?
- what did it mean?
- how should immediate workflow outcome be judged?

Those belong to Evaluate.

---

## 11. Promotion rule

Promotion inside memory space may increase trust tier.

Promotion inside memory space does not by itself create:

- canon
- truth
- unrestricted authority

Any move from trusted memory into canon remains separately gated by higher boundary law.

---

## 12. Memory object closure rule

Every memory pass must:

- emit exactly one memory object

or

- explicitly recruit downward

If neither occurs, the pass is incomplete.
