# ExecutionSurface Workflow Coherence Governor

## Status

This document is a workflow companion surface for early governed recursion over
`ExecutionSurface`.

It does not replace:

- `README.ArchitectMorphogenesisWorkflow.md`
- `README.ExecutionSurfaceGovernedRecursionEntryContract.md`
- `README.ExecutionSurfaceGovernedRecursionAppendices.md`

Its role is narrower:

- classify workflow passes by coherence effect
- prevent ambiguity-hiding compression from being mistaken for lawful release
- connect coherence inflation directly to hold / rebound / downgrade posture

---

## Purpose

The existing workflow already governs:

- single active subject
- single active role
- declared state transitions
- unresolved tensions
- hold / rebound / downgrade behavior

What was still missing was a compact enforcement grammar for this question:

**Did a role pass reduce ambiguity lawfully, or merely smooth it into something
that looks coherent enough to release?**

This file exists to make that distinction explicit.

---

## Core Invariant

No workflow pass may increase apparent subject coherence beyond what is justified
by:

- the active packet
- the active subject state
- the active role law
- the still-live unresolved tensions

If a pass exceeds that boundary, it loses release authority even if the prose
looks elegant.

---

## Non-Goals

This governor does not:

- ban synthesis
- ban compression
- ban clean writing
- require permanent tension-holding
- treat every simplification as workflow failure

It is not an anti-clarity doctrine.
It is an anti-false-resolution doctrine.

---

## Classification Ladder

Workflow passes should be classified as one of the following:

### WC0 - Observation Only

The pass inspects, inventories, or pressures the current subject without
compressing it.

Typical examples:

- audit findings
- review packets
- seam maps
- bounded risk identification

### WC1 - Lawful Compression

The pass reduces wording or structural redundancy while preserving live
distinctions and unresolved tensions honestly.

Typical examples:

- narrowing a subject statement
- removing duplicated language
- compressing non-claims without changing force

### WC2 - Lawful Synthesis

The pass combines previously explicit material into a smaller coherent form
without importing later-layer assumptions or hiding active tensions.

Typical examples:

- combining two already pressure-tested clauses
- moving a bounded fork into explicit later-work language
- releasing a subject only after the active asymmetry is genuinely exhausted

### WC3 - Ambiguity-Hiding Smoothing

The pass appears coherent but reduces visible ambiguity faster than the packet
or role law justifies.

Typical examples:

- turning unresolved tensions into elegant but under-grounded prose
- compressing a still-live distinction into a vague unity statement
- replacing explicit boundary language with smoother but weaker language

### WC4 - Authority-Inflating Resolution

The pass silently widens authority, imports later-layer assumptions, or resolves
questions not lawfully in scope.

Typical examples:

- releasing a subject by relying on unstated later architecture
- importing implementation, schema, or packet law into an earlier subject
- allowing a role to do the work of a different role without declaration

---

## Hard Rules

### Rule 1 - WC3 may not release a subject

A pass classified as `WC3` may at most request:

- Hold
- Rebound
- Downgrade

It may not request release.

### Rule 2 - WC4 loses write privilege automatically

If a pass is classified as `WC4`, it should automatically lose write privilege
for the current cycle and route into:

- Hold
- Rebound
- Downgrade

until the Administrator explicitly re-bounds the work.

### Rule 3 - Elegant prose does not override active tension

If the prose is cleaner but the active unresolved tension is less explicit than
before, the pass should be evaluated for `WC3` or `WC4`.

### Rule 4 - Later-layer imports count as inflation

If a pass relies on later-layer assumptions to stabilize the current subject,
that is inflation even when the imported material is plausible.

---

## Diagnostic Questions

When evaluating a workflow pass, ask:

1. Did the pass reduce ambiguity, or only hide it?
2. Were unresolved tensions preserved explicitly?
3. Did the role remain inside its own lawful function?
4. Did the pass import later-layer assumptions into the active subject?
5. Did the pass beautify or generalize beyond the packet?
6. Is the recommended next move narrower than the current subject, or only more
   polished?

If questions 1, 4, 5, or 6 raise concern, the pass should be checked for `WC3`
or `WC4`.

---

## Workflow Consequences

This governor should apply especially to:

- Constructor passes that feel too complete too early
- Reflector passes that unify faster than the subject has earned
- Auditor passes that release based on elegance rather than explicit boundary
  sufficiency
- Administrator reroutes that skip unresolved intermediate compression

It should not be used to freeze lawful progress.
It should be used to keep lawful progress honest.

---

## Relation To Existing Workflow Law

This governor does not invent new state outcomes.

It sharpens when to use existing outcomes:

- `Hold` when coherence is overstated but recoverable in place
- `Rebound` when the subject needs a narrower intermediate compression
- `Downgrade` when the pass exposed instability that belongs to a lower layer

It also sharpens when a release request is invalid even if the pass otherwise
looks clean.

---

## Failure Posture

When a pass is judged `WC3` or `WC4`, the default response is not to discard the
work automatically.

The default response is to:

- preserve the finding
- keep the helpful pressure or partial compression
- strip the false release force
- route the subject back into a lawful next step

This governor therefore protects momentum while removing false closure.

---

## One-Line Summary

The Workflow Coherence Governor classifies recursion passes by how much
coherence they add beyond packet-bounded subject law and prevents ambiguity-
hiding or authority-inflating passes from releasing subjects unlawfully.
