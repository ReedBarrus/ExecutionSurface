# ExecutionSurface Runtime Grammar Corpus

## Status

This directory defines the active runtime grammar corpus for `ExecutionSurface`.

It is a practical architecture-and-implementation support set for the active
development front.

Its job is narrower than broad DME theory restatement:

- stabilize runtime seam composition
- stabilize object-family grammar
- stabilize emission and authority posture
- stop morphogenesis and implementation from running blind against runtime reality

This corpus does not claim canon.
It does not claim truth uplift.
It does not replace executable repo reality.

## Purpose

The repo already contains:

- executable runtime code
- higher-level architecture notes
- an implementation ladder

What was missing was a compact runtime doctrine set that answers five distinct
questions:

1. where are the active seams
2. what object families exist at those seams
3. what each seam may emit
4. what authority each emitted surface does or does not carry
5. what implementation order follows from that grammar

This directory exists to answer those questions without collapsing them into one
large mixed note.

## Runtime Posture Established Here

Current active posture:

- `A1`, `A2`, `W1`, and `S1` are structural objects
- support recruitment begins at `H1`
- `An`, `SegmentTransition`, `M1`, `BN`, trajectory frames, `Q`, and `A3` are
  support objects
- receipts are subordinate audit surfaces only
- read-side projection is lawful only when it does not invent authority
- semantic overlay is not part of default runtime authority

## Update Rule

Update this corpus when one of the following changes:

- a seam boundary changes
- an object family changes
- an emission rule changes
- an authority boundary changes
- an implementation rung depends on a new runtime law

Do not widen these files into general repo inventory or broad philosophical
restatement.
