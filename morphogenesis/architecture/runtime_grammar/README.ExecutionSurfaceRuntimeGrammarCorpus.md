# ExecutionSurface Runtime Grammar Corpus

## Status

This directory defines the active runtime grammar corpus for `ExecutionSurface`.

It is a compact support set for active implementation and audit work.

Its job is to keep morphogenesis, implementation, and future projection work
anchored to executable runtime reality.

## Purpose

The runtime grammar corpus should answer these questions cleanly:

1. what object families exist in the runtime
2. what relation families connect them
3. where the commit boundary actually is
4. what each seam may emit
5. what packaging is lawful when direct object exposure is too heavy

## Runtime posture established here

Current active posture:

- direct structural/support object exposure is the default
- typed refs are the first fallback
- bounded object cards or compact projections are later fallbacks
- receipts are subordinate audit surfaces only
- semantic overlay is outside default runtime authority
- direct support-object exposure is mandatory, with packaging allowed only as a
  subordinate fallback
- the substrate is best understood as an attributed temporal support topology,
  not a flat receipt host
- the substrate function stack is layered, with corpus/index access subordinate
  to temporal and neighborhood truth

## Update rule

Update this corpus when one of the following changes:

- an object family changes
- a relation family changes
- the commit boundary changes
- an emission rule changes
- a packaging fallback becomes durable enough to deserve formal law

Do not widen these files into broad philosophical restatement.

## One-line summary

This corpus keeps ExecutionSurface grounded in its actual runtime topology:
object families, relation families, commit-boundary law, emission law, and the
bounded fallbacks allowed when direct object exposure is too heavy.
