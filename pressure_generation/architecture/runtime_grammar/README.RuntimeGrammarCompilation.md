## Runtime grammar in one pass

### 1. Object law

`ExecutionSurface` is direct-object-first.

The active runtime families are:

- structural objects through `S1`
- support objects beginning at `H1`
- committed `MemoryObject` as the projected singular admission envelope
- read-side projections as bounded convenience surfaces
- receipts as subordinate audit surfaces only

### 2. Emission law

Default exposure order is:

1. direct object
2. typed ref
3. bounded object card
4. compact derived projection
5. subordinate receipt

Direct support-object exposure is mandatory wherever honest carriage remains
practical.

### 3. Commit-boundary law

The current implementation still stores mixed `H1` / `M1` directly.

The frozen target is:

- lawful support payload admitted into a singular `MemoryObject` envelope

That envelope must:

- bind lawful payload
- preserve payload-native dimensionality
- expose typed relation hooks without flattening payload

### 4. Substrate topology

The active substrate model is three-layer:

1. temporal support graph / hypergraph
2. neighborhood / basin complex
3. corpus / index access layer

Layer 1 is primary truth.

Layer 2 organizes retained continuity.

Layer 3 provides access without replacing lower-layer truth.

### 5. Intrinsic substrate functions

The substrate itself is responsible for:

- admission
- persistence
- provenance linkage
- temporal placement
- support composition retention
- continuity constraints
- neighborhood organization
- recurrence support
- comparison support
- reconstruction support
- addressability
- retrieval

### 6. Over-substrate functions

These remain real but secondary:

- query projection
- reconstruction emission
- comparative reports
- workbench integration
- LM staging
- human-facing projection
- benchmark and validator receipts

### 7. Topology posture

The runtime is not just a pipeline plus receipts.

It is:

- a structural chain before support recruitment
- a support topology after recruitment
- a commit boundary
- a post-commit temporal and neighborhood substrate
- a bounded read-side layer above it

### 8. Current remaining pressure seams

The main remaining compression-heavy seams are:

- `runtime/lm/WorkbenchLmWrapper.js`
- `runtime/CrossRunDynamicsReport.js`
- live provenance and digest scripts

These should stay clearly subordinate to direct runtime truth.

### 9. Anti-inflation rule

The runtime coherence governor should continue to fence:

- receipt-only packets treated as runtime truth
- packaging treated as ontology
- semantic overlays becoming required
- projections outranking the objects they package

## One-line summary

This compilation condenses the active runtime grammar into one upload-facing
surface: direct objects first, `MemoryObject` as the commit-boundary target, a
three-layer substrate, explicit intrinsic functions, subordinate read-side
layers, and strong anti-inflation rules.
