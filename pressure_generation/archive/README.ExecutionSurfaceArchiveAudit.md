# ExecutionSurface Morphogenesis Audit

## Status

This document audits the current `ExecutionSurface/morphogenesis/` tree after
cleanup, archival, reseeding, and debate-surface extraction.

It is an audit and transition surface.

It does not itself replace runtime grammar authority, workflow history, or the
future reseeded architecture surfaces.

## Purpose

The purpose of this audit is to sort current morphogenesis material into three
classes:

- `conserved mutatable`
- `mixed mutatable`
- `quarantine/archive`

This split exists because the repo now has a much cleaner runtime basis than the
older morphogenesis trunk was operating from.

We should not:

- drag stale descriptor-first doctrine forward
- let old workflow scaffolding masquerade as live authority
- throw away useful architectural pressure that can still inform reseeding

## Classification vocabulary

### Conserved mutatable

Keep active.

These files reflect current runtime truth closely enough that they should remain
live and continue evolving.

### Mixed mutatable

Mine and rewrite.

These files contain useful subject matter, constraints, or future-facing ideas,
but their framing, object language, or process assumptions are no longer clean
enough to remain active authority as-is.

### Quarantine/archive

Freeze as history.

These files are mainly process residue, stale packets, or old state traces.
They may retain historical value, but should not steer current runtime or
implementation work directly.

## Audit summary

The audit result is now:

- the `runtime_grammar/` corpus is the conserved core
- the live architecture seed, live subject registry, and live implementation
  ladder have been reseeded around that core
- the prior workflow and support band now lives in explicit archive
- one future-debate surface preserves later translation material without
  remaining active authority

## Immediate cleanup sequence

The completed sequence was:

1. freeze the `quarantine/archive` set as historical
2. extract conserved ideas into reseeded authority surfaces or the future-debate
   holding surface
3. create a new architecture seed aligned to:
   - `MemoryObject`
   - substrate topology
   - substrate function ledger
   - direct exposure law
4. create a new subject registry aligned to the new substrate basis
5. create a new implementation ladder aligned to topology detection, admission,
   and bounded proof
6. move consumed support files to archive once their salvage was rehoused

## Recommended archive posture

Archive should preserve history, not continue authority.

The archived set should remain readable, but should be marked as:

- historical morphogenesis process material
- not current runtime authority
- not the current implementation front

That freeze is now represented physically under:

- `morphogenesis/archive/`

## One-line summary

The `ExecutionSurface/morphogenesis/` tree now splits cleanly into a conserved
runtime grammar core, a smaller band of mixed documents worth mining and
rewriting, and a historical recursion/process band that should move to archive
once the new seed, registry, and ladder are in place.
